Vocab Games v23 · Design Specification

A React/Vite browser-based language learning app · lingolearn-hue.github.io/vocab-games

This document describes the architecture, data formats, component design, and UX patterns of the Vocab Games application. It is maintained alongside the codebase and updated as features are added. Each chapter is independently collapsible; only one section at a time is expanded.

1. Overview

Vocab Games is a mobile-first Progressive Web App for vocabulary and grammar learning, supporting Chinese, Japanese, German, Spanish, and English. It uses a lightweight spaced-repetition system and organises content into three game groups:

GroupGames
Vocabulary DrillsFlashcard, Race Car, Pair Match, Typing
Language in ContextGap Fill, Graded Reader, Dialogue
Grammar DrillsGrammar Patterns, Matching Drills (Gender · Tone · Measure Words)

2. Data Formats

All content is stored as static JSON under public/ and fetched at runtime. No build-time imports of data files — all fetches use relative paths with ./ prefix (required for GitHub Pages subdirectory deployment).

2.1 Vocab List

File: public/vocab/{lang}-en.json

Keys are declared as an array so entries can be stored as compact arrays (not objects), reducing file size significantly for large lists.

{
  "id": "zh-en",
  "language": "zh",
  "native": "en",
  "keys": ["entry", "reading", "translation", "pos", "categories", "level", "gender", "measureWord"],
  "entries": [
    ["苹果", "píngguǒ", ["apple"], "noun", ["food", "fruit"], "HSK1", null, "个"],
    ["水",   "shuǐ",    ["water"], "noun", ["food", "drinks"], "HSK1", null, "杯"]
  ]
}
FieldTypeNotes
entrystringThe foreign word. German includes article: "der Apfel"
readingstring | nullPinyin (zh) or furigana (ja). Null for de/es/en
translationstring[]Always an array; first element is primary
posstring | nullnoun · verb · adj · adv · phrase · conj
categoriesstring[]Semantic tags: ["food","fruit"]
levelstring | nullHSK1–6 · N5–N1 · A1–C2
gender"m"|"f"|"n"|nullde: m/f/n · es: m/f · others: null
measureWordstring | null个/只/冊 etc. zh/ja nouns only

The loader (vocab.js:loadList) normalises entries into objects with an id field ("{listId}::{index}").

2.2 Sentences (Gap Fill)

File: public/sentences/{lang}-en.json

Each sentence contains its own answer options embedded with {correct/distractor/distractor} syntax. The engine picks the first {...} group as the blank.

{
  "language": "zh",
  "native": "en",
  "sentences": [
    {
      "text": "我每天{吃/喝/买}苹果。",
      "translation": "I eat apples every day.",
      "level": "HSK1"
    },
    {
      "text": "随着科技的{发展/衰退/停止}，生活越来越方便。",
      "translation": "With the development of technology, life becomes more convenient.",
      "level": "HSK4"
    }
  ]
}

The parseFixedSentence(text) function in vocab.js returns { before, answer, distractors, after }. Options are shuffled before display.

2.3 Grammar Patterns

File: public/grammar/{lang}-en.json

Three exercise types. All three share id, level, category, title, explanation, type.

Type: fill-blank 
{
  "id": "zh-g-001",
  "level": "HSK1",
  "category": "measure-words",
  "title": "Measure word 个 (gè) — general",
  "explanation": "个 is the most common measure word, used for people and general objects.",
  "type": "fill-blank",
  "template": "我有一___ {[朋友|学生|苹果]}。",
  "hint": "个 (gè) is used with people and general objects.",
  "distractors": ["个", "只", "本", "张"]
}

The template field uses ___ as the blank position. The optional {[a|b|c]} syntax selects a random word for the wildcard slot in each instantiation.

Type: pick-correct 
{
  "id": "de-g-033",
  "level": "B1",
  "category": "conjunctions",
  "title": "weil vs denn",
  "explanation": "weil → verb-last. denn → verb-second (main clause).",
  "type": "pick-correct",
  "sentences": [
    { "text": "Ich bleibe zuhause, weil ich krank bin.", "correct": true },
    { "text": "Ich bleibe zuhause, denn ich bin krank.", "correct": true },
    { "text": "Ich bleibe zuhause, weil ich bin krank.", "correct": false,
      "error": "Nach weil kommt Verb-Letzt: weil ich krank bin." }
  ]
}

2–4 sentences; user identifies all correct ones. Wrong answers show error feedback.

Type: tile-order 
{
  "id": "zh-g-028",
  "level": "HSK5",
  "category": "particles",
  "title": "被 passive with agent",
  "explanation": "被 construction marks subject as receiver. Agent + VP follows.",
  "type": "tile-order",
  "tiles": ["这扇窗户", "被风", "吹开了"],
  "answers": [
    { "order": [0, 1, 2], "note": null },
    { "order": [1, 0, 2], "note": "Agent-first variant also acceptable." }
  ]
}

Multiple valid orderings supported. First answer is the canonical one shown on incorrect.

Grammar SRS scores are stored in localStorage['grammarScores'] as { [patternId]: { attempts, correct, streak } }. No time-based scheduling — occurrence-based priority queue only.

3. Application State

All shared state lives in AppContext.jsx. No external state library. Games read from context and write via scoreActions.

3.2 SRS Score Record

 
// Per vocabulary entry, stored in localStorage['vocabScores']
{
  global: 'unseen' | 'learning' | 'mastered',
  racecar:   { score: 0–5, dueIn: 0+ },
  pairmatch: { score: 0–5, dueIn: 0+ },
  flashcard: { score: 0–5, dueIn: 0+ },
  gapfill:   { score: 0–5, dueIn: 0+ },
  typing:    { score: 0–5, dueIn: 0+ },
}

Spacing: dueIn is decremented by 1 for every other entry shown (tickAll). When a correct answer is given, dueIn is set to SPACING[score] = [0, 3, 6, 9, 12, 15]. When dueIn > 0, the entry is deprioritised in srsPick.

Auto-master: When all 5 game scores reach 5, global is promoted to 'mastered'. Manual reset via Vocab Browser ↩ button.

Due for review (Stats screen): entries where average game score < 3.5 across all games.

4.2 Flashcard
┌─────────────────────────────────────────┐ │ ← Back ⚙️ 3 / 847 │ │ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │ │ │ │ ┌───────────────────────────────┐ │ │ │ 苹果 🔊 │ │ ← FRONT FACE │ │ píngguǒ │ │ ← ruby reading │ │ │ │ │ │ ● ● ○ ○ ○ │ │ ← SRS score dots │ └───────────────────────────────┘ │ │ [flip animation on tap] │ │ ┌───────────────────────────────┐ │ │ │ 苹果 píngguǒ 🔊 │ │ ← BACK FACE │ │ ─────────────────────────── │ │ │ │ apple 🔊 │ │ │ └───────────────────────────────┘ │ │ │ │ [✗ Unknown] [⭐ Master] [✓ Known] │ └─────────────────────────────────────────┘

Gestures: Swipe ← = Unknown · Swipe → = Known · Swipe ↑ = Master · Swipe ↓ = Detail panel · Tap/Space = flip. Swipe is locked to dominant axis (no diagonals). Left/right always allowed regardless of flip state.

Flip animation: CSS rotateY on a .fc-card-flip wrapper with transform-style: preserve-3d. New cards get noAnim class for 100ms to prevent reverse-flip on mount.

Detail panel: opens on swipe-down or ↓ key. Shows all translations, editable mnemonic, TextWithLookup of the entry text.

4.3 Race Car
┌────────────────────────────────────────────┐ │ ← Back 苹果 = ? 0 🏁 0× │ │ │ │ │ │ │ │ │ │ 吃饭 │ 苹果 │ │ ← falling tiles │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ apple │ │ ← correct tile │ ─┼───┼─────────┼─────────┼─ boost zone ─ │ ← zone line (blue tint) │ │ │ │ 🚗 │ │ ← car in normal zone │ │ × │ ─ ─ ─ ─ ─ ─ ─ │ │ ← normal zone bottom └──┴───┴─────────┴─────────┴─────────────────┘

Zones: Bottom 15% = normal zone (default car position). 15–25% from bottom = boost zone (blue divider line, "boost" label at right). Above 25% = inactive for car movement.

Touch: Pointer events on the entire .rc-lanes div. Car position updated directly via carRef.current.style (bypasses React state for 60fps smoothness). carLane state updated only when lane changes (for collision fallback). Boost toggled when finger enters boost zone.

Speed formula: BASE_SPEED × defaultSpeed (slider, 0.5–2.0) × boostMultiplier × streakMultiplier

Collision: Uses pixel coordinates from carPosRef and lanesRef.getBoundingClientRect(). Tile horizontal bounds = lane × laneWidth + 4px to + laneWidth - 8px.

Tile sizing: CJK tiles with ruby annotation get min-height: 88px and data-has-ruby="true". Font uses clamp() based on language and text length.

4.5 Graded Reader

Two modes: library (passage list with tag filters) and reading (passage text + EN toggle).

Uses TextWithLookup for all text. The lookup engine (reader.js:buildLookup) builds a longest-match trie from activeEntries. Tapping a highlighted word opens WordPopup showing translation, SRS status, mnemonic, and 🔊 speak button.

Series are grouped in the library with a ▸ expand button. Tags shown as filter chips grouped by type.

Bubble colour is assigned per speaker index from a palette of 6 colours. After a question is answered it collapses to a single-line badge; tapping toggles the feedback explanation. Filter chips on list screen: Type (observe/choice), Level, Topic.

4.7 Grammar Trainer

Two-screen flow: type selector → infinite stream.

Type selector shows three cards: Fill the Blank · Pick Correct · Word Order. Each card shows pattern count and accuracy percentage. Selecting a type enters an infinite stream of patterns of that type only, sorted by priority (unseen first, then lowest accuracy). At the end of each cycle the pool re-sorts.

Filter bar (always visible): level chips + category chips. Selecting a type resets the stream index.

4.8 Matching Drills

Three sub-drills, selected from a drill list screen:

DrillLanguagesMechanic
Genderde (der/die/das) · es (el/la)Article stripped from entry; pick correct article
ToneszhPer-syllable tone identification from pinyin diacritics. Multi-syllable: each syllable drilled separately with context
Measure Wordszh · jaPick correct measure word for noun. Distractors from vocab pool + fallback list

All three use the same useQuiz hook: priority queue by accuracy, infinite cycle, SRS scores under gender: · tone: · mw: prefixes in grammarScores.

Card shows: large entry character (2.8rem) + 🔊 speak button + reading/ruby (1.1rem) + translation (muted).

5.2 Dark Mode

Applied via document.documentElement.setAttribute('data-theme', 'dark'|'light').

  • auto: reads prefers-color-scheme media query
  • light: forces [data-theme="light"] blocks (explicitly white background, black text)
  • dark: forces [data-theme="dark"] blocks

Every CSS file has both @media (prefers-color-scheme: dark) and [data-theme="dark"] blocks, and explicit [data-theme="light"] override blocks to cancel the media query when user has forced light mode on a dark-system device.

Light mode base: #fff background, #111 primary text, #444–#555 secondary text. No light grey text below #555 in light mode.

  setOpen(prev => prev === id ? null : id)

9. Known Limitations & Future Work

9.1 Current Limitations
  • Built-in vocab lists are very small (20–38 words). App becomes significantly more useful with the user's own large vocab files dropped into public/vocab/
  • English content is minimal (1 dialogue, 5 grammar patterns, reading-focused only)
  • Grammar SRS is occurrence-based only — no time-based scheduling
  • No URL routing — browser back button does nothing; not linkable to specific screens
  • Web Speech API TTS may not be fully local on Android/Chrome (may use Google servers)
  • color-mix() CSS in dialogue bubbles may not render in older Safari — consider fallback

9.2 Potential Future Features
  • Time-based SRS scheduling — show "due today" cards based on days since last seen, not just occurrence spacing
  • Keyboard shortcuts — number keys 1/2/3 for option selection in Gap Fill, Grammar, Matching
  • Audio recording — record yourself repeating a word; compare waveform (browser MediaRecorder API)
  • User vocab import — drag-and-drop CSV/JSON import from Settings
  • Grammar session complete — show summary card after cycling all patterns in a session
  • Capacitor wrapper — for App Store distribution (minimal changes needed from current PWA)

10. Adventure Mode
Adventure Mode is a campaign-based learning path alongside the free-play mode. Currently implemented for Japanese (N5 campaign). Accessed via the ⚔️ button on the home screen; returns to free play via ← Free Play in the adventure header.

10.3 Chapter Phase Flow
Story Intro ↓ 📚 Vocab Phase • Preview word list (entry + reading + translation + 🔊) • Launch: Flashcard / Pair Match / Race Car (filtered to chapter words) • "Continue to Grammar →" ↓ 📐 Grammar Phase • Collapsible pattern cards (fill-blank / pick-correct) • Each card has an inline exercise; ✓ marks done • "📖 Grammar Dictionary" button → searchable reference • "Continue to Dialogue →" ↓ 💬 Dialogue Phase • Same WhatsApp-style dialogue as main Dialogue game • observe and choice types both supported • "Continue to Reading →" unlocked when all turns answered ↓ 📖 Passage Phase • TextWithLookup reader passage • EN toggle, 🔊 speak button • "Complete Chapter →" ↓ ⭐ Complete Phase • Grammar artifact reward displayed • Story outro text • Next Chapter / ← Map

Phase progress is persisted to localStorage['adventureProgress'] after every advance. Returning to a chapter resumes at the saved phase. Phases are soft — you can always tap Continue without finishing the exercises.

10.4 Chapter Map

Vertical scroll, displayed in reverse order (highest chapter at top, Chapter 1 at bottom — direction of upward progression). Chapter cards show:

  • Chapter number, Japanese title, English subtitle
  • Level badge (N5, N4…) + grammar artifact name
  • Status: locked 🔒 / active (phase dots) / complete ✓
  • The next incomplete chapter is highlighted with a blue glow border

Unlock logic: Chapter 1 always unlocked. Each subsequent chapter unlocks when the previous is marked complete. Soft — no score threshold required.

10.5 Grammar Dictionary

A standalone searchable reference screen accessible from:

  • Adventure chapter grammar phase (shows chapter patterns starred)
  • Setup home screen (future: add to Grammar Drills group)
  • Direct route: screen = 'grammar-dict'

Filters: full-text search, level chips, category chips. When opened from a chapter, an additional "★ This chapter" filter shows only the current chapter's patterns. Chapter patterns are marked with ★ and a gold border.

Pattern cards are collapsible and show: explanation + worked example (template/sentences/tiles depending on type).

11. Dialogue TSV Format

New dialogue content is authored in TSV (tab-separated values) files under public/dialogues/tsv/. The full specification is at public/dialogues/tsv/FORMAT_SPEC.html. Summary below.

11.1 File Naming
PatternMeaning
advLLCC.tsvAdventure campaign — level LL, chapter CC. e.g. adv0102.tsv
scnNN.tsvStandalone scene set — all languages. e.g. scn01.tsv

11.2 Column Order

 
@columns	id	speaker	cmd	en	zh	ja	de	es
ColNameNotes
1idadv010201-04a — prefix + level + chapter + dialogue + line + branch suffix
2speakernarrator · NPC name · player · question · option
3cmdcorrect on answer options; reserved for future jump/flag commands
4–8en zh ja de esOne column per language. Empty = two tabs. English always present as translation.

11.3 Key Rules
  • Target language only — text is displayed in the active language. English is revealed on tap, never automatically shown.
  • Branching — player lines with matching number + different suffix (04a, 04b) are choice buttons. NPC responses carry the same suffix. A line without suffix after a branch always displays (convergence).
  • Questions — question line followed by option lines with same number and a/b/c suffixes. Option with cmd=correct is the right answer. Wrong answers loop back to the question.
  • Sort order — lines displayed by ID: numeric part ascending, then suffix a→b→c.
  • Difficulty scope — adventure dialogues must stay within the vocabulary and grammar unlocked by the current chapter and all prior chapters.
