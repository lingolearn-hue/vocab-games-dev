Dialogue TSV Format Specification

Vocab Games · Adventure & Scene Dialogues · v1.0

1. Overview

Dialogues are stored as tab-separated value (TSV) files. Each file contains one or more dialogues for a chapter or scene set, in all five supported languages simultaneously. The format is designed to be edited directly in a spreadsheet application or plain text editor.

Design principle: all dialogue text is in the target language by default. Translation to the native language (English) is available on demand via a tap/click on each line — it is never shown automatically. This keeps the learner immersed in the target language.

2. File Naming

PatternMeaningExample
advLLCC.tsvAdventure campaign, level LL, chapter CCadv0102.tsv — Level 01, Chapter 02
scnNN.tsvStandalone scene set NN (all languages)scn01.tsv — Scene set 01

Files live under public/dialogues/tsv/. The existing JSON files under public/dialogues/ remain for legacy content and are not replaced.

3. File Structure

# adv0102 — Adventure Campaign, Level 01, Chapter 02: First Mission
# Comment lines start with #. Ignored by parser.

@columns	id	speaker	cmd	en	zh	ja	de	es

# ── Dialogue 1 title comment ─────────
adv010201-01	guild_master		Welcome!	欢迎！	ようこそ！	Willkommen!	¡Bienvenido!
…

# ── Dialogue 2 title comment ─────────
adv010202-01	narrator		…
A @columns header line must appear exactly once, before any data rows. It defines the column order for the entire file.

4. Column Specification

#NameRequiredDescription
1idYesUnique line identifier — see §5
2speakerYesSpeaker role — see §6
3cmdNoCommand / flag — see §7. Leave empty (two tabs) if unused.
4enYesEnglish — always present; used as native-language translation
5zhNoChinese (Mandarin, simplified)
6jaNoJapanese
7deNoGerman
8esNoSpanish
Empty language cells are represented by two consecutive tab characters (an empty column). The parser skips empty cells gracefully — a line without a target-language value is not displayed for that language.

Language columns may be extended in the future (Korean, French…) by appending columns and updating the @columns header. No other changes are needed.

5. ID Format

adv010201-03a
└─┬─┘└┬┘└┬┘└┬┘└┬┘ └┬─┘└┬┘
  │   │  │  │  │    │   └── Branch suffix: a, b, c … (optional)
  │   │  │  │  │    └────── Line number within dialogue: 01, 02 …
  │   │  │  │  └─────────── Dialogue number within chapter: 01, 02 …
  │   │  │  └────────────── Chapter number: 01, 02 …
  │   │  └───────────────── Level number: 01, 02 …
  │   └──────────────────── Campaign/set identifier digits
  └──────────────────────── Prefix: adv (adventure) or scn (scene)
Ordering rule: lines are displayed in ascending ID order — numerically by line number, then alphabetically by suffix (a → b → c). The file may be written in any order; the parser always sorts before rendering.

7. Command Column (cmd)

Currently only one value is used. The column is reserved for future jump and flag commands.

ValueUsed onEffect
correctoption lineMarks this option as the correct answer to the preceding question. Exactly one option per question block must carry this flag.
(empty)anyNo command. Leave two consecutive tabs.
Future reserved values (not yet implemented): jump:ID to skip to a specific line, set:FLAG to record a story flag, if:FLAG to conditionally show a line.

8. Branching Logic

Branches are encoded entirely through the ID suffix. No explicit markup is needed.

Player choice branches

When two or more consecutive player lines share the same line number but differ in suffix (04a, 04b), they are rendered as choice buttons. The player picks one.

NPC lines that follow a branch carry the matching suffix: 05a is the NPC response to choice 04a, 05b responds to 04b. A line without a suffix after a branch means the conversation has converged — it is always shown regardless of which branch was taken.

adv010201-04a	player		Let's go together.	…
adv010201-04b	player		I prefer to go alone.	…

adv010201-05a	emma		Great! I know a shortcut.	…   ← only shown if player chose 04a
adv010201-05b	emma		Be careful out there.	…    ← only shown if player chose 04b

adv010201-06	guild_master		Good luck.	…           ← always shown (no suffix)

Question / answer blocks

A question line is followed immediately by option lines. Options share the same line number as each other (e.g. all are line 07), differentiated by suffix (a, b, c). The option with cmd=correct is the right answer. On a wrong answer, the question is re-displayed. On a correct answer, the dialogue advances to the next line.

adv010201-06q	question		クリスタルは何ですか？	…
adv010201-07a	option	correct	光のクリスタル	The Crystal of Light	…
adv010201-07b	option		ギルドの剣	The guild's sword	…
adv010201-07c	option		村人の子ども	The villager's child	…
The q suffix on the question line is optional but recommended for clarity. Any non-numeric suffix that is not a–z is treated as a label, not a branch.

9. Translation (Tap to Reveal)

All text in every line — including questions and answer options — is written in the target language. A small 🌐 button on each line reveals the English translation on tap. It is never shown automatically.

Line typeTarget language showsTap reveals
NPC / narratorTarget language textEnglish column
Player choice optionTarget language option textEnglish option text
Question promptTarget language questionEnglish question
Answer optionTarget language answerEnglish answer
The English column is the only column ever used as a translation. It is never the primary display language (unless the learner is studying English, in which case another language serves as the translation).

10. Language Scope & Difficulty

Critical for adventure mode: each dialogue is tagged to a chapter level. All text in that dialogue must stay within the vocabulary and grammar scope unlocked by that chapter and all previous chapters. A player who has just completed Chapter 1 (は・です・か・も, greetings, professions) should be able to read every line in that chapter's dialogue without encountering unknown grammar or vocabulary. Simpler is always better than authentic.

Practical guidelines per JLPT level:

N5 Ch 1–38–10 wordsOnly grammar introduced in that chapter and priorOnly words from chapters 1 through current
N5 Ch 4–810–15 wordsBuilding grammar compounding allowedChapter vocabulary + common N5 only
N5 Ch 9–1215–20 wordsFull N5 grammarFull N5 vocabulary

10b. Writing for Absolute Beginners

The most common mistake when writing adventure dialogues is writing natural-sounding language rather than level-appropriate language. These are very different things at N5/A1. A line that sounds perfectly natural to a native speaker may contain three unknown grammar points and five unknown words for a chapter-1 learner. Simpler is always better.

The core rule

Every word and every grammar structure in a dialogue line must appear in the chapter's vocabulary list or grammar patterns, or in any previous chapter. If a word is not in the list, cut it or replace it.

Concrete before / after — N5 Chapter 1

❌ Too hard✓ Level-appropriateWhy
北の村で困ったことが起きています。不思議なことが起きているようです。 村で、変なことがあります。 Original uses ようです (N4), 困った (not in ch1 vocab), 起きている (te-form, not yet taught)
あなたの最初のミッションです。村へ行って、何が起きているか調べてください。 村へ行ってください。 か indirect question and 調べる are N4+. Ch1 only needs simple requests.
私はギルドマスターのレオンだ。よろしく。 わたしはレオンです。ギルドマスターです。 だ is plain form (style mismatch for early learners). Split compound sentences.

Chapter 1 sentence checklist

  • Maximum one clause per sentence — no て-form chaining, no から/ので, no けど
  • Only です/ます form — no plain form, no だ
  • No relative clauses (の, という, など)
  • No indirect speech, no reported speech
  • Subject always explicit (わたしは, これは) — no ellipsis
  • 5–8 words per sentence maximum
  • Every noun and verb must be in the chapter word list

Progression guideline

N5 Ch 3–48–12 words+ movement verbs, て-form (go and do)村へ行って、子どもを探します。
N5 Ch 5–610–15 words+ adjectives, たい, ているあの子は小さくて、かわいいです。

12. Parser Notes (for implementation)

  • Lines starting with # are comments — skip entirely
  • Lines starting with @columns define column names — parse once
  • Blank lines are ignored
  • Columns are separated by a single tab character (\t)
  • Sort all data rows by ID before processing: numeric part ascending, then suffix alphabetically
  • A player line with a suffix is a choice option; without a suffix it is a statement (display only)
  • Branch state persists within a single dialogue playthrough; resets on replay
  • The active language is determined by activeLanguage from app context — pick the corresponding column
  • If the target-language cell is empty, skip rendering that line for that language (do not show an empty bubble)
